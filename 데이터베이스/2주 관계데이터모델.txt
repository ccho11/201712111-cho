무결성 제약조건

키key
	특정 투플을 식별할때 사용하는 속성 혹은 속성의 집합
	키가 되는 속성은 반드시 값이 달라서 투플을 구별할 수 있어야 함
	키는 릴레이션 간의 관계에도 사용
	키는 투플 식별장치 릴레이션간의 연결고리
슈퍼키super key
	고유속성
	위의 조건을 만족하면 모두 슈퍼키 가능
후보키candidate key
	투플을 유일하게 식별할 수 있는 속성의 최소 집합
	효율성 측면에서 중요함
기본키Primary key,pk
	여러 후보키 중 하나를 선정하여 대표로 삼는 키
	특성을 고려해서 선택
		고유한값인가
		null제외
		변동이없는가
		최대한 적은 수의 속성인가
		향후 문제의 소지가 없는가
대리키surrogate key
	일련번호와 같은 가상의 속성을 만들어 기본키로사용 >> 대리키 인조키
	임의의 값 의미X	
대체키alternate key
	기본키로 선정X 후보키
외래키Foreign Key,fk
	릴레이션간의 관계표현
	다른 기본키를 참조 도메인은 서로같아야함
	기본키값변경=외래키값변경
	null 과 중복값허용
	자기자신의 기본키참조가능 외래키가 기본키가능

무결성 제약조건
	데이터 무결성
		데이터베이스에 저장된 데이터의 일관성과 정확성을 지키는 것을 말함
	도메인 무결성 제약조건(도메인 제약)
		릴레이션내 투플들이 속성의 도메인에 지정된 값만 가져야함
		SQL문에서 데이터 형식type 널null/not null 기본값default 체크check 사용하여 지정
	개체 무결성 제약조건(기본키 제약)
		릴레이션은 기본키를 지정하고 그에 따른 무결성 원칙
		기본키는 null 값을 가져서는 안 되며 릴레이션 내에 오직 하나의 값만 존재해야 한다는 조건
	참조 무결성 제약조건(외래키 제약)
		릴레이션 간의 참조 관계를 선언하는 제약조건
		자식 릴레이션의 외래키는 부모 릴레이션의 기본키와 도메인이 동일해야함, 
		자식 릴레이션의 값이 변경될 때 부모 릴레이션의 제약을 받음

수행
	개체 무결성 제약조건
		기본키가 유일한지 null아닌지
		삽입-기본키 값이 같으면 금지
		수정-기본키 값이 같거나 null로도 수정금지
		삭제-특별한 확인이 필요하지 않으며 즉시 수행함
	참조 무결성 제약조건
		삽입-부모키가 없으면 자식키에 사용할 수 없음
		삭제-부모키에서 투플삭제할경우 고려
			>>즉시 작업중지
			>>자식키관련 투플 삭제
			>>초기설정된 다른 값으로 변경
			>>null값으로 설정
  명령어  |                          의미			|	예
restricted	자식키에서 참조중일 경우 부모키 삭제 작업 거부	  요청한 작업 삭제중지(에러 처리)
cascade	자식키의 관련 투플을 같이 삭제		  자식키의 해당 투플을 같이 연쇄적으로 삭제
default	자식키의 관련 투플을 미리 설정해둔 값으로 변경	  기본값으로 변경(미리 설정한 값, default)
null	자식키의 관련 투플을 null로 설정(null 허가할때)	  null값으로 변경
