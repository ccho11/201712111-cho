기본개념
 공유데이터	특정조직의 여러 사용자가 함꼐 소유하고 이용할 수 있는 공용 데이터
 통합데이터	최소의 중복과 통제 가능한 중복만 허용하는 데이터
 저장 데이터	컴퓨터가 접근할 수 있는 매체에 저장된 데이터
 운영 데이터	조직의 주요 기능을 수행하기 위해 지속적으로 꼭 필요한 데이터

특징
 실시간 접근	사용자의 데이터 요구에 실시간으로 응답해야함
 계속 변화	데이터의 계속적인 삽입 삭제 수정을 통해 현재의 정확한 데이터를 유지
 동시공유		여러 사용자가 서로 다른 데이터의 동시 사용뿐만 아니라 같은 데이터의 동시 사용도 지원
 내용기반참조	데이터가 저장된 주소나 위치가 아닌 내용(값)으로 참조 ex) 재고량이 1,000개 이상인 제품의 이름을 검색하시오
데이터와 데이터베이스

데이터의 형태에 따른 분류 
	정형데이터, 반정형데이터, 비정형데이터
	-정형데이터(structured data) 구조화된 데이터, 즉 미리 정해진 구조에 따라 저장된 데이터 ex) 엑셀 스프레드시트, 관계 데이터 베이스의 테이블
	-반정형데이터 구조에따라 저장된 데이터 내용 안에 구조에 대한 설명이 존재 구조를 파악하는 파싱pasing 과정필요 보통파일형태 저장 ex)html, xml, json 문서나 웹로그, 센서데이터
	-비정형데이터 정해진 구조가없이 저장된 데이터 ex)소셜 데이터 텍스트, 영상, 이미지, 워드나 PDF문서등의 멀티미디어 데이터 facebook insta twitter youtube 등

파일 시스템 
 오래 전부터 사용된 정보 처리 시스템 데이터를 파일로 관리하기위해 생성삭제수정검색등의 기능제공 sw(software) 운영체제와 함께 설치 응용프로그램마다 별도의 파일로 관리
 문제점 같은 내용의 데이터 여러파일에 중복저장됨. 응용 프로그램이 데이터 파일에 종속적. 데이터 파일에 대한 동시 공유보안회복 기능부족 응용 프로그램의 개발이 어려움

데이터베이스 관리 시스템 DBMS(data base management system)
 기능
	정의기능 구조를 정의 수정
	조작기능 삽입삭제수정검색의 연산가능
	제어기능 항상정확하고 안전하게 유지
 장단점
  장점	중복통제 독립성확보 동시공유 보안향상 무결성유지 표준화 장애회복 응용프로그램개발비용감소
  단점	많은비용 백업과회복이불편 중앙집중관리로취약점존재
