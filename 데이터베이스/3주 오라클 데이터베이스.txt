데이터 타입
char	size 크기의 고정 길이 문자 데이터 타입 
(size)	최대/최소 2000/1 바이트
	정해진 길이만큼의 저장영역

varchar2 	size 크기의 가변 길이 문자 데이터 타입 
(size)	최대/최소 4000/1 바이트	
	입력될 데이터 길이의 편차가 심한 데이터의 타입으로 지정

nvarchar2  국가별 문자 집합에 따른 size 
(size)	크기의 문자 또는 바이트의 가변길이 문자 데이터 타입
	최대/최소 4000/1 바이트

number	정밀도p 와 스케일s 로 표현되는 숫자 데이터 타입
p, s	p 1~38 s -84~127
	수치 데이터 저장에 사용
	NUMBER(정밀도, 스케일)
	정밀도 : 소수점을 포함한 전체자리수
	스케일 : 소수점 이하 자리수를 지정
	   >>생략하면 소수점 이하 반올림되어 정수값만 저장됨
	number(5)
12345	|1 |2 |3 |4 |5 |
	number(7,2)
12345.67	|1 |2 |3 |4 |5. |6 |7 |


DATE	날짜 형식을 저장하기 위한 데이터 타입
	별다른 설정없으면 년, 월, 일만 출력됨
	YY/MM/DD >> 2011년 12월 15일 = 11/12/15
미국식	DD/Mon/YY >> 2011년 12월 15일 = 15/Dec/11

ROWID 	테이블 내 행의 고유 주소를 가지는 64진수 문자타입 해당 6바이트(제한된 로아이디) 또는 10바이트(확장된 로아이디)

BLOB	대용량의 바이너리 데이터를 저장하기 위한 데이터 타입 최대4GB

CLOB	대용량의 텍스트 데이터를 저장하기 위한 데이터 타입 최대4GB

BFILE	용량의 바이너리 데이터를 파일 형태로 저장하기 위한 데이터 타입 최대4GB

timestamp (n)	date 데이터 타입의 확장된 형태
		n은 milli second 자리수로 최대 9자리까지 표현가능

interval year to 	년과 월을 이용하여 기간을 저장
month		

interval day to 	일 시 분 초를 이용하여 기간을 저장
second		두 날짜값의 정확한 차이를 표현하는데 유용
